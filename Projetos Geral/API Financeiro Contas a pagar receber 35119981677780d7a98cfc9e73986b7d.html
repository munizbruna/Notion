Sobre o projeto

A função desta parte da API é gerenciar as Contas a Pagar e Receber, aqui teremos consumo/relacionamento com Pedidos, pois estes que serão contas a receber do processo, assim a API deve consultar o pedidos e lançar corretamente essa informação.

Para os DTOs verifique quais são as informações que retornam na listagem, são todas as informações do Financeiro?

Na página de detalhes devem ser todas as informações, assim como na criação, exceto o ID.

O PUT precisa do ID, mas não o altera.

O DELETE, também precisa do ID mas não deve coletar nenhum outro dado.

Itens de ação

  • Criar FinanceiroController (VALIDAR CORRETAMENTE OS DTOs)
  • GET (Listar),
  • POST (Criar)
  • PUT (Atualizar).
  • DELETE (Excluir).
