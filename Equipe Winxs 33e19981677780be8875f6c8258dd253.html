Resumo do Projeto

  • O que será feito?

O projeto consiste no desenvolvimento de um Enterprise Resource Planning (ERP) projetado para centralizar e otimizar os pilares fundamentais da operação empresarial: Estoque, Vendas, Financeiro e Recursos Humanos. Ao unificar essas frentes em uma plataforma única e intuitiva, o sistema elimina silos de informação, permitindo uma gestão de cadastros de clientes e fornecedores muito mais assertiva e um controle de inventário rigoroso, essencial para a saúde operacional do negócio.

O grande diferencial estratégico reside na automação inteligente do fluxo de trabalho, onde cada transação de venda dispara, em tempo real, a atualização do estoque e o lançamento imediato no módulo financeiro. Essa integração nativa não apenas anula falhas humanas e retrabalhos, como também transforma dados brutos em inteligência de negócio, garantindo que o gestor tenha agilidade total e precisão absoluta na tomada de decisões críticas para o crescimento da empresa.

  • Padronização de Nomenclaturas :

    Banco de Dados (Models): Cliente, Entradas, Venda, Financeiro: Recebimentos e Pagamentos, Fornecedor, Produto, Funcionário, Itens Venda.

    Variáveis e Funções: Utilizaremos o padrão “QtdEstoque, Nome, Email, CPF, e etc”.

    IDs: Toda tabela terá uma chave primária simples denominada apenas como “Id”.

  • Informações que compõe as telas :

  • Cadastro:

    Campos Cliente: Inputs de texto para Nome, Telefone, Email, CPF, CEP, Número residencial e Complemento; Input tipo Date para Data de Nascimento; Input tipo Password para Criar Senha.

    Campos Fornecedor: Inputs de texto para Nome, Telefone, CNPJ, Empresa, Email e Cidade; Form Select para Estado; Input tipo Password para Criar Senha.

    Campos Funcionário: Inputs de texto para Nome, Telefone, CPF, CEP e Empresa; Form Select para Cargo; Input tipo Email; Input tipo Password para Criar Senha.

    Ação: Botão (Button) "Enviar" com destaque em Azul Royal.

  • Página de início:

    Visão Cliente: Input de busca para produtos; Form Select para filtros de Categoria e Marcas; Componente Slider para filtrar valores; Badges (etiquetas) para destacar cupons; Ícones clicáveis para Carrinho e Central de ajuda.

    Visão Fornecedor (Cadastrar produtos): Inputs de texto para Nome e Código; Input numérico para Quantidade disponível e Mínima; Textarea para Descrição; Input de Valor para Preço unitário; Botão de deletar (Remover).

    Visão Funcionário (Cadastrar Venda): Input tipo Date para data; Input de Valor para Preço e Desconto; Form Select para Status do pagamento e Status de Entrega; Input numérico para Quantidade.

    • Na tela de início do Fornecedor, deve conter:Cadastrar produtos:Inputs de texto para Nome do produto e o código do produto (Id); Inputs numéricos para a quantidade disponível e a quantidade mínima por pedido; Botão (Button) com ícone de lixeira para a opção de remover o produto; Textarea para descrição do produto; Input de valor para o valor unitário.
    • Cadastrar Venda: Input tipo Date para a Data da venda do produto; Input de valor para Preço e Desconto; Form Select (Caixa de seleção) para Nome do Fornecedor; Input de texto para Código da compra, Destino e código do cupom; Form Select para Status do pagamento (Ex: Pendente, Pago) e Status de Entrega (Ex: Em separação, Enviado); Input numérico para Quantidade de produto vendido.
    • Página do produto: Componente de Image Upload/Display para a Imagem do produto; Input numérico (Stepper) para selecionar a quantidade do produto; Texto em destaque para o valor unitário; Bloco de texto para descrição do produto e informações do fornecedor; Textarea para a central de dúvidas; Componente de Estrelas (Rating) para avaliações; Label para a marca do produto e Botão (Button) "Comprar".

    • Na tela do Funcionário deve conter: Cards informativos exibindo Horas trabalhadas e total de vendas do dia; Inputs de texto (somente leitura) para cargo atual e salário; Input numérico ou Slider para o percentual de vendas; Input tipo Time para Horas Extras; Botão (Button) para gerar e baixar o Relatório de vendas realizadas em PDF.
    • Login para clientes, Funcionário e Fornecedores:Input de texto para Identificação (Email/CPF ou CNPJ); Input tipo Password para a senha; Botão (Button) "Entrar" em Azul Royal; Checkbox para a opção "Lembrar acesso".

    • Financeiro:

      Datas e valores dos gastos.

      Parte de pagamentos contendo: data do vencimento, valor, descrição do pagamento, data do pagamento, e status do pagamento.

      Parte de recebimentos: Data de recebimento, valor, descrição do recebimento.

      Filtros: Input tipo Date (Início e Fim) para filtragem de gastos.Tabela de Pagamentos: Colunas com dados e Status Tag (visual colorido para Pago/Pendente).Tabela de Recebimentos: Listagem simples de entradas e valores.

    • RH

    Listagem: Data Table com todos os funcionários ativos. Edição: Modal (Janela flutuante) contendo Inputs para ajuste de Salário e Form Select para mudança de Cargo. Controle: Input tipo Time ou numérico para ajuste de Horas Extras e Percentual.

    Fluxo: Faça o cadastro, redirecionar para o login, se for cliente após o login vai para a página do cliente, caso seja funcionário é redirecionado para a página do funcionário, e caso seja fornecedor vai para a página do fornecedor.
